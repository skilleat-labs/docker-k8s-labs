    1-1-2. Docker 설치 및 첫 실행¶

    실습 전 SSH 접속 먼저 하세요

    이 실습부터는 명령어가 길어지기 때문에 VM 화면에서 직접 타이핑하는 것보다 호스트 터미널에서 SSH로 접속해서 복사·붙여넣기로 진행하는 것을 강력 추천합니다.

    macOS Terminal 또는 Windows PowerShell에서 아래 명령어로 접속하세요.

    ssh vboxuser@192.168.56.10

    8) VM 안에 Docker 설치¶

    공식 설치 문서: Docker Engine on Ubuntu

    VM 터미널에서 실행합니다.

    sudo apt update
sudo apt install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo $VERSION_CODENAME) stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo systemctl enable docker
sudo systemctl start docker
sudo usermod -aG docker $USER

    권한 반영을 위해 로그아웃/로그인 후 아래 확인:

    docker --version
docker run hello-world

    Docker 자동완성도 설정합니다.

    sudo apt-get install -y bash-completion
docker completion bash | sudo tee /etc/bash_completion.d/docker > /dev/null
source /etc/bash_completion.d/docker

    아래와 같은 출력이 나오면 정상입니다.

    Unable to find image 'hello-world:latest' locally
latest: Pulling from library/hello-world
...
Hello from Docker!
This message shows that your installation appears to be working correctly.

    출력 결과 해석

    ① 이미지를 로컬에서 찾지 못함

    Unable to find image 'hello-world:latest' locally

    처음 실행하는 이미지라 내 컴퓨터에 없습니다. Docker가 자동으로 Docker Hub에서 다운로드합니다.

    ② Docker가 동작하는 방식 (출력 메시지 그대로)

    1. Docker 클라이언트가 Docker 데몬에 요청
    2. 데몬이 Docker Hub에서 hello-world 이미지를 pull
    3. 이미지로 컨테이너를 생성하고 실행
    4. 실행 결과(텍스트)를 터미널에 출력

    ③ 실행 후 docker ps 로 확인하면?

    docker ps

    아무것도 나오지 않습니다. hello-world 컨테이너는 메시지를 출력하는 것이 전부라 실행이 끝나는 즉시 종료됩니다. docker ps는 현재 실행 중인 컨테이너만 보여주기 때문입니다.

    ④ 그럼 이미지는?

    docker images

    hello-world 이미지는 다운로드된 상태로 남아 있습니다. 컨테이너(실행 인스턴스)는 사라졌지만, 이미지(설계도)는 로컬에 캐시되어 있습니다.

    9) Docker로 Nginx 한 번에 실행¶

    기존 VM Nginx와 포트 충돌 방지를 위해 먼저 VM의 Nginx를 중지합니다.

    sudo systemctl stop nginx

    Docker Nginx 실행:

    docker run -d --name nginx-container -p 8081:80 nginx:latest

    터미널에서 확인:

    docker ps
curl http://192.168.56.10:8081

    브라우저에서 확인:

    http://192.168.56.10:8081

    Nginx Welcome 페이지가 보이면 성공입니다.

    10) VM 방식 vs 컨테이너 방식 체감 정리¶

    비교 포인트:

    • VM: OS 부팅이 필요하고 준비 시간이 김
    • 컨테이너: 앱 단위로 즉시 실행되며 시작이 빠름
    • VM: 리소스 사용량이 큼
    • 컨테이너: 상대적으로 가볍고 배포가 빠름

    트러블슈팅¶

    • 브라우저 접속 불가: ip addr show enp0s9 로 VM IP 확인, 192.168.56.10 이 잡혀있는지 확인
    • Nginx 미동작: sudo systemctl status nginx --no-pager 확인
    • Docker 권한 오류: 재로그인 후 groups에 docker 포함 여부 확인
    • 포트 충돌: sudo ss -lntp | grep :80 또는 docker ps로 점유 프로세스 확인
