    1-5. Dockerfile 최적화 실습¶

    실습 목표¶

    • Dockerfile 작성 시 효율성과 보안을 고려한 최적화 기법 체험하기
    • RUN 명령어 최소화, 캐시 전략, .dockerignore 활용하기
    • 멀티 스테이지 빌드와 비-root 사용자 적용 실습

    예상 소요 시간 (약 30분)¶

    • 0~5분: 기본 Dockerfile 작성 및 빌드
    • 5~15분: RUN 최소화, 캐시 전략, .dockerignore 실습
    • 15~23분: 멀티 스테이지 빌드 실습
    • 23~30분: 비-root 보안 적용 및 정리

    사전 조건¶

    • Docker 설치 완료
    • 기본 터미널 명령어 이해

    확인:

    docker --version
docker ps

    1) 기본 Dockerfile 작성 및 빌드 체험¶

    1-1. 작업 디렉토리 준비¶

    mkdir -p ~/docker-optimize
cd ~/docker-optimize

    1-2. 간단한 Node.js 앱 준비¶

    app.js 파일 생성:

    cat > app.js <<'EOF'
const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello from optimized Docker!');
});

app.listen(3000, () => {
  console.log('App running on port 3000');
});
EOF

    package.json 파일 생성:

    cat > package.json <<'EOF'
{
  "name": "docker-optimize",
  "version": "1.0.0",
  "main": "app.js",
  "scripts": {
    "start": "node app.js"
  },
  "dependencies": {
    "express": "^4.18.0"
  }
}
EOF

    1-3. 기본 Dockerfile 작성¶

    Dockerfile 생성:

    cat > Dockerfile <<'EOF'
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .

EXPOSE 3000
CMD ["npm", "start"]
EOF

    1-4. 빌드 및 실행¶

    docker build -t myapp:v1 .
docker run -d --name myapp-v1 -p 3000:3000 myapp:v1
curl http://localhost:3000

    관찰 포인트:

    • 빌드 시간과 이미지 크기 확인 (docker images)
    • 기본 Dockerfile의 구조 이해

    2) RUN 명령어 최소화 실습¶

    2-1. 비효율적인 Dockerfile 예시¶

    Dockerfile.bad 생성 (비교용):

    cat > Dockerfile.bad <<'EOF'
FROM node:18-alpine

WORKDIR /app

RUN apk add --no-cache git
RUN apk add --no-cache curl
RUN apk add --no-cache vim

COPY package*.json ./
RUN npm install

RUN mkdir -p /app/logs
RUN mkdir -p /app/cache

COPY . .

EXPOSE 3000
CMD ["npm", "start"]
EOF

    2-2. 최적화된 Dockerfile 작성¶

    Dockerfile 수정 (레이어 통합):

    cat > Dockerfile <<'EOF'
FROM node:18-alpine

WORKDIR /app

# RUN 명령어 통합 (레이어 수 최소화)
RUN apk add --no-cache git curl vim && \
    mkdir -p /app/logs /app/cache

COPY package*.json ./
RUN npm install

COPY . .

EXPOSE 3000
CMD ["npm", "start"]
EOF

    2-3. 빌드 비교¶

    # 최적화 전 이미지 크기 확인
docker build -f Dockerfile.bad -t myapp:bad .
docker images | grep myapp

# 최적화 후 이미지 크기 확인
docker build -t myapp:v2 .
docker images | grep myapp

    관찰 포인트:

    • 레이어 수 차이 (docker history myapp:v2)
    • 이미지 크기 감소 효과
    • 백슬래시(\)로 가독성 유지

    3) 캐시 전략 실습¶

    3-1. 캐시가 깨지는 비효율적인 순서¶

    Dockerfile.cache-bad 생성:

    cat > Dockerfile.cache-bad <<'EOF'
FROM node:18-alpine

WORKDIR /app

# 소스 코드 먼저 복사 (캐시 비효율)
COPY . .

RUN npm install

EXPOSE 3000
CMD ["npm", "start"]
EOF

    3-2. 최적화된 캐시 순서¶

    Dockerfile 수정 (의존성 → 소스 코드 순서):

    cat > Dockerfile <<'EOF'
FROM node:18-alpine

WORKDIR /app

# 의존성 파일 먼저 복사 (캐시 최적화)
COPY package*.json ./
RUN npm install

# 소스 코드 나중에 복사
COPY . .

EXPOSE 3000
CMD ["npm", "start"]
EOF

    3-3. 캐시 효과 체험¶

    # 첫 빌드
docker build -t myapp:v3 .
time docker build -t myapp:v3 .  # 빌드 시간 측정

# 소스 코드만 변경 후 재빌드 (캐시 효과)
echo "// comment added" >> app.js
time docker build -t myapp:v4 .  # npm install 캐시 재사용 확인

    관찰 포인트:

    • 두 번째 빌드에서 npm install 단계가 캐시에서 가져옴
    • 빌드 시간 단축 효과

    4) .dockerignore 활용 실습¶

    4-1. .dockerignore 파일 생성¶

    .dockerignore 생성:

    cat > .dockerignore <<'EOF'
# Node.js
node_modules
npm-debug.log*

# Git
.git
.gitignore

# IDE
.vscode
.idea

# OS
.DS_Store
Thumbs.db

# Logs
*.log
logs/

# Runtime
.env
.env.local

# Build artifacts
dist/
build/
EOF

    4-2. 빌드 컨텍스트 크기 비교¶

    # .dockerignore 없이 빌드 컨텍스트 크기 확인
docker build --no-cache -t myapp:v5 .

# .dockerignore 적용 후 빌드
docker build --no-cache -t myapp:v6 .

    관찰 포인트:

    • 빌드 속도 향상
    • 보안 파일(node_modules 등) 제외 효과

    5) 멀티 스테이지 빌드 실습¶

    5-1. 빌드와 실행 환경 분리¶

    Dockerfile.multi 생성:

    cat > Dockerfile.multi <<'EOF'
# 빌드 스테이지
FROM node:18-alpine AS builder

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

# 실행 스테이지 (경량화)
FROM node:18-alpine

RUN addgroup -g 1001 -S nodejs && \
    adduser -S nextjs -u 1001

WORKDIR /app

# 빌드 스테이지에서 필요한 파일만 복사
COPY --from=builder /app/node_modules ./node_modules
COPY . .

USER nextjs

EXPOSE 3000
CMD ["npm", "start"]
EOF

    5-2. 멀티 스테이지 빌드 실행¶

    docker build -f Dockerfile.multi -t myapp:multi .
docker images | grep myapp

    관찰 포인트:

    • 이미지 크기 대폭 감소 (빌드 도구 제외)
    • COPY --from=builder 구문 이해

    6) 비-root 보안 적용 실습¶

    6-1. root 위험성 체험¶

    기존 컨테이너에서 root 권한 확인:

    docker run --rm myapp:v1 whoami  # root로 실행됨

    6-2. 비-root 사용자 적용¶

    Dockerfile.secure 생성:

    cat > Dockerfile.secure <<'EOF'
FROM node:18-alpine

# 비-root 사용자 생성
RUN addgroup -g 1001 -S appgroup && \
    adduser -S appuser -u 1001 -G appgroup

WORKDIR /app

# 권한 설정
RUN chown -R appuser:appgroup /app

# 의존성 설치
COPY package*.json ./
RUN npm install

# 소스 복사
COPY . .

# 사용자 전환
USER appuser

EXPOSE 3000
CMD ["npm", "start"]
EOF

    6-3. 보안 적용 빌드 및 실행¶

    docker build -f Dockerfile.secure -t myapp:secure .
docker run --rm myapp:secure whoami  # appuser로 실행됨
docker run -d --name myapp-secure -p 3001:3000 myapp:secure
curl http://localhost:3001

    관찰 포인트:

    • 최소 권한 원칙 적용
    • K8s 등 컨테이너 오케스트레이션 호환성 향상

    7) 정리 및 비교¶

    7-1. 모든 이미지 크기 비교¶

    docker images | grep myapp

    7-2. 컨테이너 정리¶

    docker rm -f $(docker ps -aq) 2>/dev/null || true
docker rmi myapp:v1 myapp:v2 myapp:v3 myapp:v4 myapp:v5 myapp:v6 myapp:multi myapp:secure 2>/dev/null || true

    7-3. 최적화 효과 요약¶

    버전 특징 이미지 크기 빌드 시간
    v1 기본 보통 보통
    v2 RUN 최소화 감소 비슷
    v3 캐시 최적화 비슷 단축
    multi 멀티 스테이지 대폭 감소 비슷
    secure 비-root 비슷 비슷

    📝 핵심 요약¶

    1. 경량·재현 가능 이미지: RUN 통합, .dockerignore, 멀티 스테이지로 이미지 크기 최소화
    2. 캐시·멀티스테이지 최적화: 의존성 우선 복사로 빌드 캐시 활용, 빌드/실행 환경 분리로 효율성 극대화
    3. 보안은 Dockerfile부터: root 대신 비-root 사용자 적용으로 최소 권한 원칙 준수
