# 4-2. Resource requests/limits & QoS 실습

## 실습 목표

- CPU/Memory requests와 limits를 설정하고 효과를 확인한다.
- QoS Class(Guaranteed · Burstable · BestEffort)가 어떻게 결정되는지 실습한다.
- ResourceQuota와 LimitRange로 Namespace 단위 자원을 제한한다.

## 전제 조건

- `k8s-lab` 네임스페이스 사용 중

---

## 1) requests와 limits 개념 정리

| 설정 | 역할 | 초과 시 |
|---|---|---|
| `requests.cpu` | Scheduler 배치 기준 (최소 보장) | CPU throttling (강제 종료 없음) |
| `requests.memory` | Scheduler 배치 기준 (최소 보장) | - |
| `limits.cpu` | 최대 CPU 사용량 | CPU throttling |
| `limits.memory` | 최대 메모리 사용량 | **OOMKilled** (컨테이너 강제 종료) |

---

## 2) BestEffort Pod 생성 (requests/limits 없음)

`pod-besteffort.yaml` 파일을 만듭니다.

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: pod-besteffort
spec:
  containers:
    - name: app
      image: nginx:1.25
```

```bash
kubectl apply -f pod-besteffort.yaml
kubectl get pod pod-besteffort -o yaml | grep -A 5 "qosClass"
```

```
qosClass: BestEffort
```

---

## 3) Burstable Pod 생성 (requests < limits)

`pod-burstable.yaml` 파일을 만듭니다.

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: pod-burstable
spec:
  containers:
    - name: app
      image: nginx:1.25
      resources:
        requests:
          cpu: 100m
          memory: 128Mi
        limits:
          cpu: 500m
          memory: 512Mi
```

```bash
kubectl apply -f pod-burstable.yaml
kubectl get pod pod-burstable -o yaml | grep -A 5 "qosClass"
```

```
qosClass: Burstable
```

---

## 4) Guaranteed Pod 생성 (requests == limits)

`pod-guaranteed.yaml` 파일을 만듭니다.

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: pod-guaranteed
spec:
  containers:
    - name: app
      image: nginx:1.25
      resources:
        requests:
          cpu: 200m
          memory: 256Mi
        limits:
          cpu: 200m
          memory: 256Mi
```

```bash
kubectl apply -f pod-guaranteed.yaml
kubectl get pod pod-guaranteed -o yaml | grep -A 5 "qosClass"
```

```
qosClass: Guaranteed
```

### QoS 클래스 확인

```bash
kubectl get pods -o custom-columns=\
"NAME:.metadata.name,QoS:.status.qosClass,CPU-REQ:.spec.containers[0].resources.requests.cpu,MEM-REQ:.spec.containers[0].resources.requests.memory"
```

## 5) 메모리 초과 → OOMKilled 확인

메모리를 limits보다 많이 사용하는 Pod를 생성합니다.

`pod-oom.yaml` 파일을 만듭니다.

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: pod-oom
spec:
  containers:
    - name: app
      image: busybox:1.36
      command:
        - sh
        - -c
        - |
          # 50MB 메모리를 강제로 할당
          dd if=/dev/zero bs=1M count=50 | tr '\0' 'x' > /dev/null &
          sleep 3600
      resources:
        requests:
          memory: 10Mi
        limits:
          memory: 20Mi    # 20MB 제한이지만 50MB 요청 → OOMKilled
```

```bash
kubectl apply -f pod-oom.yaml
kubectl get pod pod-oom -w
```

잠시 후 `OOMKilled` 상태로 변경됩니다.

```bash
kubectl describe pod pod-oom | grep -A 5 "Last State\|Reason"
```

```
Reason: OOMKilled
```

## 6) ResourceQuota 설정 (Namespace 단위 제한)

`resourcequota.yaml` 파일을 만듭니다.

```yaml
apiVersion: v1
kind: ResourceQuota
metadata:
  name: lab-quota
spec:
  hard:
    requests.cpu: "2"
    requests.memory: 1Gi
    limits.cpu: "4"
    limits.memory: 2Gi
    count/pods: "10"
```

```bash
kubectl apply -f resourcequota.yaml
kubectl describe resourcequota lab-quota
```

### Quota 초과 시도

현재 사용 중인 리소스가 Quota를 초과하면 Pod 생성이 거부됩니다.

```bash
kubectl describe resourcequota lab-quota | grep -A 20 "Resource"
```

---

## 7) LimitRange 설정 (기본값 자동 적용)

`limitrange.yaml` 파일을 만듭니다.

```yaml
apiVersion: v1
kind: LimitRange
metadata:
  name: lab-limitrange
spec:
  limits:
    - type: Container
      default:
        cpu: 200m
        memory: 256Mi
      defaultRequest:
        cpu: 100m
        memory: 128Mi
      max:
        cpu: "1"
        memory: 1Gi
      min:
        cpu: 50m
        memory: 64Mi
```

```bash
kubectl apply -f limitrange.yaml
kubectl describe limitrange lab-limitrange
```

### 확인 — requests/limits 없는 Pod에 기본값 자동 적용

```bash
kubectl delete pod pod-besteffort
kubectl apply -f pod-besteffort.yaml
kubectl get pod pod-besteffort -o yaml | grep -A 10 resources
```

LimitRange가 적용되어 requests/limits가 자동으로 채워집니다.

## 정리 (리소스 삭제)

```bash
kubectl delete pod pod-besteffort pod-burstable pod-guaranteed pod-oom
kubectl delete resourcequota lab-quota
kubectl delete limitrange lab-limitrange
```

## requests 설정 가이드

```
requests = 평균 사용량 기준
limits    = 피크 사용량 + 20~30% 여유분

예시:
  평균 CPU: 150m → requests.cpu: 150m
  피크 CPU: 400m → limits.cpu: 500m
  평균 Memory: 200Mi → requests.memory: 200Mi
  피크 Memory: 400Mi → limits.memory: 512Mi
```

## 트러블슈팅

| 증상 | 확인 사항 |
|---|---|
| Pod `Pending` (리소스 부족) | 노드 allocatable 리소스 확인 (`kubectl describe node`) |
| Pod 계속 재시작, 이유 `OOMKilled` | `limits.memory` 증가 또는 앱 메모리 누수 확인 |
| Pod 생성 거부 `exceeded quota` | `kubectl describe resourcequota`로 현재 사용량 확인 |
