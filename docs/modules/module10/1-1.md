# 1-1. 실습 환경 설정 (Docker Desktop / Rancher Desktop)

## 실습 목표

- Docker Desktop 또는 Rancher Desktop에서 로컬 Kubernetes 클러스터를 활성화한다.
- kubectl이 클러스터에 연결되었음을 확인한다.
- 이후 모든 실습의 기반 환경을 준비한다.

## 준비물

- macOS 또는 Windows PC
- 인터넷 연결
- 관리자 권한

---

## 옵션 A — Docker Desktop으로 설정

### 1) Docker Desktop 설치

1. [Docker Desktop 공식 다운로드](https://www.docker.com/products/docker-desktop/) 페이지에서 운영체제에 맞는 버전을 설치합니다.
2. 설치 후 Docker Desktop을 실행합니다.

### 2) Kubernetes 활성화

1. 오른쪽 상단 톱니바퀴(Settings)를 클릭합니다.
2. 왼쪽 메뉴에서 **Kubernetes**를 선택합니다.
3. **Enable Kubernetes** 체크박스를 활성화합니다.
4. **Apply & Restart** 버튼을 클릭합니다.
5. 하단 상태 표시줄에 초록색 Kubernetes 아이콘이 나타날 때까지 기다립니다. (1~3분 소요)

### 3) 연결 확인

터미널에서 실행합니다.

```bash
kubectl config current-context
```

`docker-desktop`이 출력되면 정상입니다.

```bash
kubectl get nodes
```

```
NAME             STATUS   ROLES           AGE   VERSION
docker-desktop   Ready    control-plane   1m    v1.29.x
```

## 옵션 B — Rancher Desktop으로 설정

### 1) Rancher Desktop 설치

1. [Rancher Desktop 공식 다운로드](https://rancherdesktop.io/) 페이지에서 설치 파일을 다운로드합니다.
2. 설치 후 실행합니다.
3. 초기 설정 화면에서 아래를 선택합니다.
   - **Container Runtime**: `dockerd (moby)` — Docker 명령어 호환
   - **Kubernetes Version**: 최신 안정 버전 (예: v1.29.x)

### 2) 연결 확인

```bash
kubectl config current-context
```

`rancher-desktop`이 출력되면 정상입니다.

```bash
kubectl get nodes
```

```
NAME                   STATUS   ROLES                  AGE   VERSION
lima-rancher-desktop   Ready    control-plane,master   2m    v1.29.x
```

## 공통 확인 사항

### kubectl 버전 확인

```bash
kubectl version --client
```

### 클러스터 전체 상태 확인

```bash
kubectl cluster-info
```

```
Kubernetes control plane is running at https://127.0.0.1:6443
```

### 노드 상세 정보 확인

`-o wide` 옵션을 붙이면 기본 출력보다 더 많은 정보를 한눈에 볼 수 있습니다.

```bash
kubectl get nodes -o wide
```

```
NAME             STATUS   ROLES           AGE   VERSION   INTERNAL-IP    OS-IMAGE         KERNEL-VERSION   CONTAINER-RUNTIME
docker-desktop   Ready    control-plane   5m    v1.29.x   192.168.65.3   Docker Desktop   6.6.x            docker://26.x.x
```

기본 `kubectl get nodes`와 비교해보세요. `-o wide`에서 추가로 보이는 항목은 다음과 같습니다.

| 추가 항목 | 설명 |
|---|---|
| `INTERNAL-IP` | 노드의 내부 IP 주소 |
| `OS-IMAGE` | 노드에서 사용 중인 OS |
| `KERNEL-VERSION` | 리눅스 커널 버전 |
| `CONTAINER-RUNTIME` | 컨테이너 실행 엔진 (docker, containerd 등) |

> `-o wide`는 `kubectl get pods`, `kubectl get nodes` 등 대부분의 리소스에 사용할 수 있습니다. 이후 실습에서도 자주 활용합니다.

## (선택) AKS 연결 — 추후 사용

로컬 환경에 익숙해진 후 Azure AKS로 전환할 때는 아래 명령어로 컨텍스트를 추가합니다.

```bash
az aks get-credentials --resource-group <rg이름> --name <aks이름>
kubectl config get-contexts        # 등록된 클러스터 목록 확인
kubectl config use-context <이름>  # 클러스터 전환
```

## 트러블슈팅

| 증상 | 확인 사항 |
|---|---|
| `kubectl: command not found` | Docker Desktop 재설치 또는 PATH 설정 확인 |
| `Unable to connect to the server` | Docker Desktop이 실행 중인지, Kubernetes가 활성화되었는지 확인 |
| 노드 상태 `NotReady` | 1~2분 후 재확인, 또는 Docker Desktop 재시작 |
| Rancher Desktop 노드 없음 | Rancher Desktop 앱이 완전히 시작될 때까지 대기 |
