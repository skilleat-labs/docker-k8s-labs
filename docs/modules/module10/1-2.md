# 1-2. kubectl 기본 명령어 & 클러스터 구조 탐색

## 실습 목표

- kubectl의 핵심 명령어 패턴을 익힌다.
- Control Plane과 Worker Node 구성 요소를 직접 확인한다.
- `kube-system` 네임스페이스에서 K8s 내부 컴포넌트를 탐색한다.

## 전제 조건

- 1-1 실습 완료 (로컬 클러스터 실행 중)

## 1) kubectl 기본 명령어 패턴

kubectl 명령어는 `kubectl <동사> <리소스> [이름]` 형태입니다.

| 동사 | 설명 |
|---|---|
| `get` | 리소스 목록 또는 상세 조회 |
| `describe` | 리소스 상세 이벤트/상태 조회 |
| `apply -f` | YAML 파일로 리소스 생성/업데이트 |
| `delete` | 리소스 삭제 |
| `logs` | Pod 컨테이너 로그 확인 |
| `exec` | Pod 내부 명령어 실행 |
| `port-forward` | 로컬 포트를 Pod/Service에 포워딩 |

---

## 2) 노드 탐색

```bash
kubectl get nodes
kubectl get nodes -o wide   # IP, OS, Container Runtime 포함
kubectl describe node docker-desktop
```

`describe` 출력에서 확인할 항목:

- `Roles`: control-plane 여부
- `Capacity` / `Allocatable`: CPU·메모리 가용량
- `Conditions`: 노드 상태 (Ready, MemoryPressure 등)
- `System Info`: K8s 버전, Container Runtime 버전

---

## 3) kube-system 네임스페이스 탐색

Control Plane 컴포넌트는 `kube-system` 네임스페이스에서 Pod로 실행됩니다.

```bash
kubectl get pods -n kube-system
```

> `-n kube-system`은 **네임스페이스를 지정**하는 옵션입니다. 네임스페이스는 뒤에서 자세히 배우지만, 지금은 "K8s 내부 시스템 Pod들이 모여있는 별도 공간" 정도로 이해하면 됩니다.

주요 Pod 목록:

| Pod 이름 패턴 | 역할 |
|---|---|
| `kube-apiserver-*` | API Server — 모든 요청의 진입점 |
| `etcd-*` | etcd — 클러스터 상태 저장소 |
| `kube-scheduler-*` | Scheduler — Pod 배치 결정 |
| `kube-controller-manager-*` | Controller Manager — 상태 유지 |
| `kube-proxy-*` | kube-proxy — 네트워크 규칙 |
| `coredns-*` | CoreDNS — 클러스터 내부 DNS |

```bash
kubectl describe pod kube-apiserver-docker-desktop -n kube-system
```

## 4) 리소스 단축 이름(alias)

자주 쓰이는 단축 이름을 익혀둡니다.

```bash
kubectl get po          # pods
kubectl get deploy      # deployments
kubectl get svc         # services
kubectl get cm          # configmaps
kubectl get ns          # namespaces
kubectl get pv          # persistentvolumes
kubectl get pvc         # persistentvolumeclaims
kubectl get hpa         # horizontalpodautoscalers
```

---

## 5) 출력 형식 옵션

```bash
kubectl get pods                   # 기본 테이블
kubectl get pods -o wide           # 확장 정보 (IP, Node)
kubectl get pods -o yaml           # YAML 전체 출력
kubectl get pods -o json           # JSON 출력
kubectl get pods --show-labels     # 라벨 포함
```

---

## 6) 네임스페이스 전체 조회

> `-n`, `-A` 옵션은 모두 네임스페이스와 관련된 옵션입니다. 네임스페이스는 뒤에서 자세히 배웁니다. 지금은 아래 명령어를 실행해서 출력이 어떻게 다른지만 확인합니다.

```bash
kubectl get pods -A                # 모든 네임스페이스의 Pod를 한 번에 조회
kubectl get pods --all-namespaces  # 위와 동일
```

`-A` 옵션을 붙이면 `NAMESPACE` 컬럼이 추가되어 어느 공간의 Pod인지 함께 표시됩니다.

## 7) 실습 — API Server에 직접 요청

kubectl은 내부적으로 K8s REST API를 호출합니다. 프록시를 띄워 직접 확인합니다.

```bash
kubectl proxy &
```

새 터미널에서:

```bash
curl http://localhost:8001/api/v1/pods
```

Pod 목록이 JSON으로 반환됩니다. (아직 아무것도 없으면 빈 `items` 배열)

프록시 종료:

```bash
fg
# Ctrl+C
```
