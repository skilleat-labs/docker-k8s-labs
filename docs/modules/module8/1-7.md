## 1) Docker Compose 개념 이해

Docker Compose는 여러 컨테이너를 하나의 YAML 파일로 정의하고, 한 번에 실행/제어하도록 돕는 도구입니다.

`docker-compose.yml`의 핵심 최상위 키:

- `services`: 실행할 컨테이너 목록
- `networks`: 서비스 간 네트워크 연결 정의
- `volumes`: 데이터를 유지할 볼륨 정의

> 참고: 최신 Docker Compose(v2)에서는 `version:` 키를 생략해도 됩니다.

## 2) 실습 디렉토리 생성

Compose 파일은 반드시 별도 폴더에서 시작합니다.

```bash
mkdir ~/compose-lab && cd ~/compose-lab
```

!!! info "폴더 이름 = Compose 프로젝트 이름"
    Docker Compose는 `docker-compose.yml`이 있는 **폴더 이름을 프로젝트 이름**으로 자동 사용합니다.

    프로젝트 이름은 컨테이너, 네트워크, 볼륨 이름의 **앞에 prefix로 붙습니다.**

    예를 들어 폴더 이름이 `compose-lab`이면:

    | 리소스 | 실제 생성되는 이름 |
    |--------|------------------|
    | web 컨테이너 | `compose-lab-web-1` |
    | db 컨테이너 | `compose-lab-db-1` |
    | frontend 네트워크 | `compose-lab_frontend` |
    | db-data 볼륨 | `compose-lab_db-data` |

    이 덕분에 **같은 Compose 파일을 다른 폴더에서 실행해도 충돌 없이** 독립적으로 동작합니다.

## 3) 간단한 Compose 파일 작성

다음 예제를 `docker-compose.yml` 파일로 저장합니다.

```yaml
services:
  web:
    image: nginx:alpine
    ports:
      - "8080:80"
    networks:
      - frontend

  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: example
    volumes:
      - db-data:/var/lib/mysql
    networks:
      - backend

networks:
  frontend:
  backend:

volumes:
  db-data:
```

### 설명

- `web` 서비스: Nginx 웹 서버를 8080 포트로 노출
- `db` 서비스: MySQL을 `db-data` 볼륨과 함께 실행
- `networks`: `frontend`, `backend` 2개 네트워크를 정의
- `volumes`: `db-data` 볼륨을 생성해 DB 데이터를 보존

## 4) Compose 실행

```bash
docker compose up -d
```

실행 후 확인:

```bash
docker compose ps
docker compose logs --tail=20
```

웹 브라우저에서 `http://localhost:8080`으로 접속하면 Nginx 기본 페이지가 보입니다.

---

## 5) 서비스 연결 확인

Compose는 같은 네트워크에 있는 서비스 이름을 DNS 이름으로 사용합니다.

```bash
docker compose exec web ping -c 3 db
```

이 명령어가 성공하면 `web` 컨테이너에서 `db` 컨테이너에 올바르게 연결된 것입니다.

---

## 6) 네트워크와 볼륨 확인

```bash
docker network ls
docker volume ls
```

Compose로 만든 네트워크와 볼륨 이름은 일반적으로 현재 디렉터리 이름을 접두사로 사용합니다.

```bash
docker network inspect $(docker network ls --filter name=frontend -q)
```

---

## 7) 정리

Compose 실행을 멈추고 리소스를 정리합니다.

```bash
docker compose down
```

데이터 볼륨까지 제거하려면:

```bash
docker compose down -v
```

## 8) 추가 실습 아이디어

- `web` 서비스에 `Dockerfile`을 만들어 직접 이미지 빌드해 보기
- `depends_on`을 사용해 실행 순서를 지정해 보기

- `restart: unless-stopped`를 추가해 컨테이너 자동 재시작 설정해 보기
