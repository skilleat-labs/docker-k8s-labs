# 모듈 8 개요 — VM과 컨테이너 실습

## 학습 목표

이 모듈에서는 **가상머신(VM)과 컨테이너의 차이를 직접 체감**하고, Docker의 핵심 개념과 실무 사용법을 익힙니다.

## 왜 컨테이너인가?

### 기존 방식의 문제

개발팀이 신규 서버에 애플리케이션을 배포할 때마다 겪는 일들:

- "제 로컬에서는 되는데요?" — 환경 불일치
- Nginx 버전 충돌로 기존 서비스 장애 발생
- OS 패치 한 번에 3시간짜리 배포 롤백

### 컨테이너가 해결하는 것

| 문제 | VM 방식 | 컨테이너 방식 |
|------|---------|-------------|
| 환경 불일치 | 서버마다 수동 세팅 | 이미지에 환경 포함 |
| 기동 시간 | 수분 | 수초 |
| 자원 효율 | OS 전체 부팅 | 프로세스 격리 |
| 배포 단위 | 서버 전체 | 앱 단위 |

## 실습 환경

| 항목 | 내용 |
|------|------|
| 플랫폼 | Windows / macOS |
| 가상화 도구 | VirtualBox 7.x |
| Guest OS | Ubuntu 24.04 LTS |
| 컨테이너 런타임 | Docker Engine 26.x |
| 실습 방식 | 로컬 환경에서 직접 설치·비교 |

## 단원 구성

| 단원 | 주제 | 실습 내용 |
|------|------|---------|
| 1-1 | VM vs 컨테이너 비교 | VirtualBox, Ubuntu, Nginx, Docker 직접 설치·비교 |
| 1-2 | Docker 기본 명령어 | run, exec, ps, logs, rm 동작 원리 이해 |
| 1-3 | Docker Hub & 이미지 | pull, push, tag, 레지스트리 활용 |
| 1-4 | 볼륨(Volume) | 컨테이너 데이터 영속성 실습 |
| 1-5 | Dockerfile 최적화 | 레이어 캐시, 멀티스테이지 빌드 |
| 1-6 | Docker 네트워크 | bridge, host, overlay 네트워크 실습 |
| 1-7 | Docker Compose | 멀티 컨테이너 앱 정의 및 운영 |

## 선수 지식

- Linux 기본 명령어 (ls, cd, cat, chmod)
- 웹 서버 개념 (HTTP 요청/응답)
- 네트워크 기초 (IP, 포트)
