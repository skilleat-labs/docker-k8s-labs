# 1-1-3. 이미지는 불변하다 — 직접 확인해보기

컨테이너 안에서 파일을 아무리 수정해도, **이미지 자체는 바뀌지 않습니다.**
새 컨테이너를 띄우면 항상 원래 상태로 돌아옵니다. 직접 체감해 봅시다.

## Step 1. 컨테이너 안에 들어가서 페이지 수정

```bash
docker exec -it nginx-container bash
```

접속하면 프롬프트가 아래처럼 컨테이너 ID로 바뀝니다.

```
root@a1b2c3d4e5f6:/#
```

컨테이너 쉘 안에서 Nginx 기본 페이지를 수정합니다.

```bash
echo "<h1>I changed this</h1>" > /usr/share/nginx/html/index.html
```

수정이 끝나면 컨테이너에서 나옵니다.

```bash
exit
```

프롬프트가 다시 VM 터미널로 돌아오면 정상입니다.

## Step 2. 변경된 내용 확인

터미널 또는 브라우저에서 확인합니다.

```bash
curl http://192.168.56.10:8081
```

`<h1>I changed this</h1>` 가 보이면 성공입니다. 브라우저에서 `http://192.168.56.10:8081` 로 접속해도 됩니다.

## Step 3. 컨테이너 삭제 후 새로 실행

```bash
docker rm -f nginx-container
docker run -d --name nginx-container -p 8081:80 nginx:latest
```

## Step 4. 다시 확인

`http://192.168.56.10:8081` 을 새로고침합니다.

!!! success "결과"
    **원래 Nginx 기본 페이지로 돌아옵니다.**

    컨테이너 안에서 수정한 내용은 그 컨테이너가 삭제되는 순간 사라집니다.
    `nginx:latest` **이미지는 처음부터 끝까지 변하지 않았습니다.**

!!! info "이게 왜 중요한가?"
    - 컨테이너는 이미지를 **읽기 전용**으로 사용하고, 변경사항은 별도의 레이어에만 기록됩니다.
    - 컨테이너를 삭제하면 그 변경 레이어도 같이 사라집니다.
    - 덕분에 어떤 환경에서 몇 번을 실행해도 **항상 동일한 상태**로 시작할 수 있습니다.
    - 데이터를 영구적으로 저장하려면 **볼륨(Volume)** 을 써야 합니다. → 1-4에서 다룹니다.
