# 1-6-2. 도전 과제 : 미니 블로그

## 과제 개요

지금까지 배운 Docker 네트워크 개념을 총동원해서, 컨테이너만으로 동작하는 미니 블로그 서비스를 직접 구성합니다.

브라우저에서 글을 작성하면 frontend가 backend로 요청을 보내고, backend가 데이터를 JSON 파일로 저장한 뒤 다시 응답하는 구조입니다. 모든 컨테이너는 직접 `docker run` 명령으로 실행합니다.

---

## 서비스 구조

```
사용자 브라우저
     ↓ (localhost:8080)
frontend (Nginx)
     ↓ (HTTP 요청 /posts)
backend (Flask)
     ↓
data.json 저장
```

- **frontend**: Nginx 기반 정적 페이지 서빙 + backend로 API 요청 프록시
- **backend**: Flask 기반 REST API, 게시글을 `data.json`에 저장/조회
- **두 컨테이너는 사용자 정의 네트워크를 통해 통신**

---

## 사용 이미지

| 서비스 | 이미지 |
|--------|--------|
| frontend | `skilleatlab.azurecr.io/frontend:v4-kb5` |
| backend | `skilleatlab.azurecr.io/backend:v4-kb5` |

---

## 과제 단계

### 1단계: 전용 네트워크 생성

frontend와 backend가 서로 이름으로 통신할 수 있도록 사용자 정의 네트워크를 먼저 생성합니다.

네트워크 이름은 자유롭게 지정해도 됩니다.

---

### 2단계: backend 컨테이너 실행

다음 조건을 모두 만족하도록 backend 컨테이너를 실행합니다.

- 이미지: `skilleat/backend:v4-kb5`
- 컨테이너 이름: **반드시 `backend-service`**
- 1단계에서 생성한 네트워크에 연결
- 내부 포트: **5000번** (호스트에는 노출하지 않아도 됩니다)

!!! info "왜 backend를 먼저 실행하나요?"
    frontend는 시작할 때 backend-service라는 이름으로 연결을 시도합니다.
    backend가 먼저 실행되어 있어야 frontend가 정상적으로 작동합니다.

---

### 3단계: frontend 컨테이너 실행

다음 조건을 모두 만족하도록 frontend 컨테이너를 실행합니다.

- 이미지: `skilleat/frontend:v4-kb5`
- backend와 **동일한 네트워크**에 연결
- 호스트 포트 **8080** → 컨테이너 포트 **80** 으로 바인딩

---

### 4단계: 웹 접속 및 동작 확인

브라우저 또는 curl로 아래 주소에 접속합니다.

```
http://localhost:8080
```

- 블로그 페이지가 정상 표시되는지 확인합니다.
- 글 작성 폼에 제목과 내용을 입력하고 등록합니다.
- 등록한 글이 화면에 출력되는지 확인합니다.

---

### 5단계: 추가 검증

**backend 내부 JSON 파일 확인**

작성한 글이 backend 컨테이너 내부의 `data.json`에 실제로 저장되었는지 확인합니다.
컨테이너 내부로 직접 접속해서 파일 내용을 출력해보세요.

**네트워크 소속 확인**

frontend와 backend가 동일한 네트워크에 속해 있는지 네트워크 상세 정보로 확인합니다.
