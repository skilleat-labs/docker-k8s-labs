    1-3-1. 나만의 이미지 만들고 ACR에 올리기¶

    실습 목표¶

    • Dockerfile을 직접 작성해서 나만의 이미지를 만들어보기
    • 나만의 ACR(Azure Container Registry)을 생성하기
    • 만든 이미지를 내 ACR에 push하기

    예상 소요 시간 (약 30분)¶

    • 0~10분: Dockerfile 작성 및 이미지 빌드
    • 10~18분: 내 ACR 생성
    • 18~28분: 이미지 push 및 확인
    • 28~30분: 정리

    1) Dockerfile로 나만의 이미지 만들기¶

    1-1. 작업 디렉토리 준비¶

    mkdir ~/myapp && cd ~/myapp

    1-2. 커스텀 HTML 파일 작성¶

    nano index.html

    아래 내용을 직접 입력하세요:

    <!DOCTYPE html>
<html>
  <head><meta charset="UTF-8"><title>My Docker App</title></head>
  <body>
    <h1>안녕하세요!</h1>
    <p>내가 직접 만든 Docker 이미지입니다.</p>
  </body>
</html>

    저장: Ctrl + O → Enter → 종료: Ctrl + X

    1-3. Dockerfile 작성¶

    nano Dockerfile

    아래 내용을 직접 입력하세요:

    FROM nginx:alpine
COPY index.html /usr/share/nginx/html/index.html

    저장: Ctrl + O → Enter → 종료: Ctrl + X

    구조 확인:

    ls -la ~/myapp/

    파일 구조:

    myapp/
├── Dockerfile
└── index.html

    1-4. 이미지 빌드¶

    docker build -t myapp:v1.0 .

    예상 출력:

    [+] Building ...
 => [internal] load build definition from Dockerfile
 => [1/2] FROM nginx:alpine
 => [2/2] COPY index.html /usr/share/nginx/html/index.html
 => exporting to image
Successfully built ...
Successfully tagged myapp:v1.0

    이미지 확인:

    docker images | grep myapp

    1-5. 빌드한 이미지로 컨테이너 실행 및 확인¶

    docker run -d --name myapp-test -p 9090:80 myapp:v1.0
curl http://localhost:9090

    예상 출력:

    <!DOCTYPE html>
<html>
  <head><meta charset="UTF-8"><title>My Docker App</title></head>
  ...

    내가 작성한 HTML이 그대로 나오면 성공입니다.

    # 확인 후 컨테이너 정리
docker rm -f myapp-test

    2) Python 앱 Dockerfile 패턴¶

    2-1. Python FastAPI 앱 구조 이해¶

    평가에서 제공되는 스켈레톤 코드는 아래와 같은 구조입니다.

    order/
├── app/
│   └── main.py        ← FastAPI 앱 (수정 금지)
├── requirements.txt   ← 의존성 목록 (수정 금지)
└── Dockerfile         ← 직접 작성

    order/ 디렉토리 안에서 docker build 를 실행하므로, app/main.py 는 빌드 컨텍스트 기준으로 app/main.py 경로입니다.

    2-2. 연습용 앱 준비¶

    mkdir ~/fastapi-practice && cd ~/fastapi-practice
mkdir app

    app/main.py 생성 (평가 시에는 이미 제공됩니다):

    cat > app/main.py <<'EOF'
from fastapi import FastAPI

app = FastAPI()

@app.get("/health")
def health():
    return {"status": "ok"}

@app.get("/orders")
def orders():
    return [{"id": 1, "item": "노트북"}, {"id": 2, "item": "마우스"}]
EOF

    requirements.txt 생성:

    cat > requirements.txt <<'EOF'
fastapi==0.111.0
uvicorn==0.30.1
EOF

    2-3. Dockerfile 작성¶

    nano Dockerfile

    저장: Ctrl+O → Enter → 종료: Ctrl+X

    FROM python:3.11-slim

WORKDIR /app

# requirements.txt 먼저 복사 → pip install 레이어 캐시 활용
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 앱 소스 복사
COPY app/ app/

EXPOSE 8001
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8001"]

    CMD 형식 설명

    uvicorn app.main:app 은 app/ 디렉토리 안의 main.py 파일에서 app 객체를 실행한다는 의미입니다. 포트는 서비스마다 다르게 지정합니다 (Order: 8001, Product: 8002, Notification: 8003).

    2-4. 빌드 및 /health 확인¶

    docker build -t order-service:v1 .
docker run -d --name order-test -p 8001:8001 order-service:v1

    /health 엔드포인트가 응답하는지 확인합니다.

    curl http://localhost:8001/health

    예상 출력:

    {"status": "ok"}

    curl http://localhost:8001/orders

    예상 출력:

    [{"id": 1, "item": "노트북"}, {"id": 2, "item": "마우스"}]

    2-5. 정리¶

    docker rm -f order-test
docker rmi order-service:v1

    4) 나만의 ACR 만들기¶

    2-1. ACR 이름 정하기¶

    ACR 이름은 전 세계에서 유일해야 하고, 영문 소문자와 숫자만 사용할 수 있습니다.

    예시: skilleat + 팀번호 + 이니셜

    skilleatteam01kim
skilleatteam02lee

    ACR 이름에 하이픈(-)은 사용 불가입니다.

    2-2. ACR 생성¶

    # ACR_NAME에 본인 이름으로 변경
ACR_NAME="skilleat본인이름"

az acr create \
  --resource-group skilleat-container-lab \
  --name $ACR_NAME \
  --sku Basic \
  --location koreacentral

    예상 출력:

    {
  "name": "skilleat본인이름",
  "loginServer": "skilleat본인이름.azurecr.io",
  "location": "koreacentral",
  "sku": { "name": "Basic" },
  ...
}

    2-3. ACR 생성 확인¶

    CLI로 확인:

    az acr list --resource-group skilleat-container-lab --output table

    Azure Portal에서도 확인:

    1. portal.azure.com 접속
    2. 상단 검색창에 본인 ACR 이름 입력
    3. Container Registry 항목으로 이동 → 상태가 Ready인지 확인

    5) 내 ACR에 이미지 push하기¶

    3-1. ACR 로그인¶

    az acr login --name $ACR_NAME

    예상 출력:

    Login Succeeded

    3-2. 이미지에 ACR 주소 태그 붙이기¶

    Docker는 이미지 이름의 앞부분을 레지스트리 주소로 인식합니다. myapp:v1.0만으로는 어느 레지스트리에 올릴지 알 수 없으므로, 내ACR주소/myapp:v1.0 형태로 태그를 붙여 목적지를 지정합니다.

    docker tag myapp:v1.0 $ACR_NAME.azurecr.io/myapp:v1.0

    태그 확인:

    docker images | grep $ACR_NAME

    예상 출력:

    skilleat본인이름.azurecr.io/myapp   v1.0   ...

    3-3. ACR에 push¶

    docker push $ACR_NAME.azurecr.io/myapp:v1.0

    예상 출력:

    The push refers to repository [skilleat본인이름.azurecr.io/myapp]
...
v1.0: digest: sha256:... size: ...

    3-4. ACR에 올라간 이미지 확인¶

    az acr repository list --name $ACR_NAME --output table

    특정 이미지 태그 확인:

    az acr repository show-tags --name $ACR_NAME --repository myapp

    6) 내 ACR에서 pull 해보기¶

    이미지를 삭제하고 내 ACR에서 다시 받아봅니다.

    # 로컬 이미지 삭제
docker rmi myapp:v1.0
docker rmi $ACR_NAME.azurecr.io/myapp:v1.0

# 내 ACR에서 pull
docker pull $ACR_NAME.azurecr.io/myapp:v1.0

# 실행 확인
docker run -d --name myapp-from-acr -p 9090:80 $ACR_NAME.azurecr.io/myapp:v1.0
curl http://localhost:9090

    7) 실습 정리(삭제)¶

    # 컨테이너 삭제
docker rm -f myapp-from-acr 2>/dev/null || true

# 로컬 이미지 삭제
docker rmi myapp:v1.0 2>/dev/null || true
docker rmi $ACR_NAME.azurecr.io/myapp:v1.0 2>/dev/null || true

    핵심 정리¶

    단계 명령어 의미
    이미지 빌드 docker build -t 이름:태그 . Dockerfile로 이미지 생성
    ACR 생성 az acr create ... 나만의 프라이빗 레지스트리 생성
    태그 붙이기 docker tag 원본 ACR주소/이름:태그 ACR 주소를 이미지에 연결
    push docker push ACR주소/이름:태그 내 ACR에 이미지 업로드
    pull docker pull ACR주소/이름:태그 내 ACR에서 이미지 다운로드
